# Design Direction

## Current Visual Direction

The workspace is intentionally moving toward a darker EveryCircuit-style studio interface rather than a bright card-based education dashboard.

## Current UI Shape

- compact dark header
- left sidebar for examples, lessons, and component symbols
- middle sidebar for circuit details, simulation notes, and inspector content
- right stage for waveform monitor, circuit board, and bottom status controls
- dark board with neon-like active paths and component states

## User Preferences Captured

- prefer a dark circuit-lab style over the previous light UI
- use real circuit-style icons instead of text-heavy controls
- remove large side gutters and oversized pinned header behavior
- keep scrolling natural instead of locking oversized UI at the top
- add scope-like waveform visuals
- show animated microscopic motion for energized circuits

## Interaction Direction

- keep builder approachable for beginners
- preserve drag-and-drop plus click-to-wire workflow
- make the builder feel closer to a serious lab playground than a static tutorial site
- prefer compact controls and dense but readable panes

## Styling Guardrails

- avoid returning to the earlier white shaded UI for the builder
- avoid generic SaaS card styling in the main workspace
- keep the board visually dominant
- keep symbol icons and active simulation feedback visible at a glance
