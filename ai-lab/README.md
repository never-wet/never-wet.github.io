# Cortex Lab

Cortex Lab is a browser-based AI workspace built with React, TypeScript, and Vite. It combines an Obsidian-inspired connected knowledge system with a 3D force-directed graph, a visual neural network builder, and in-browser TensorFlow.js training.

## What it includes

- A real 3D floating graph using `react-force-graph-3d` and Three.js
- Node dragging, orbit controls, zoom, hover, click focus, highlighting, and category filters
- A flowchart-style neural network builder using `@xyflow/react`
- Browser-side model training and testing using TensorFlow.js
- Notes, backlinks, tags, datasets, experiments, models, and result snapshots
- Local-first persistence with `localStorage`, IndexedDB snapshots, and IndexedDB model saves
- Import/export of workspace JSON
- Compact memory files under `src/memory/` so future assistants can orient quickly

## HTML entry points

- `dev.html` is the Vite source entry used during local development.
- `index.html` is the published static entry generated by `npm run build`.

That means opening the project folder directly should use `index.html`, while Vite development opens `dev.html`.

## How the app works

### 3D graph

The graph uses `react-force-graph-3d` for force simulation, dragging, orbit controls, and rendering. Node visuals are customized with Three.js sphere meshes plus sprite labels, while the data model comes from the workspace store.

### Builder

The builder uses a sequential validation pass to ensure the trainable path has exactly one input, one output, no cycles, and one clear forward chain. That validated layer plan is what the TensorFlow.js engine converts into a browser model.

### Training

Training presets generate lightweight demo datasets such as XOR, spiral classification, and sine regression. Completed runs save model weights into IndexedDB and create linked model/result nodes in the workspace graph.

### Persistence

Zustand persists the main workspace state to `localStorage`. A debounced IndexedDB snapshot mirrors the workspace payload, and completed TensorFlow.js models are saved with the `indexeddb://` backend.
